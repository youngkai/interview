# 分布式系统常用算法：一致性哈希、限流算法、消息重试机制

## 30 秒回答

分布式系统常见算法包括一致性哈希、限流算法和消息重试机制。一致性哈希用于缓存和分片路由，减少节点增减时的数据迁移；限流算法用于保护系统，常见有固定窗口、滑动窗口、漏桶、令牌桶；消息重试机制用于处理临时失败，但必须配合幂等、退避、最大次数和死信队列，避免无限重试拖垮系统。

## 一致性哈希

### 解决什么问题

普通取模：

```text
node = hash(key) % N
```

当节点数 N 变化时，大量 key 的映射都会变化，缓存会大面积失效。

一致性哈希把节点和 key 都映射到一个 hash 环上：

```text
key 顺时针找到第一个节点
```

节点新增或删除时，只影响环上一小段 key。

## 一致性哈希应用场景

### 1. 分布式缓存

把不同 key 路由到不同缓存节点。

### 2. 分片路由

根据 order_id、tracking_no、user_id 路由到不同分片。

### 3. 负载均衡

同一个用户或订单尽量路由到同一个实例，提高缓存命中率。

## 虚拟节点

如果每台真实机器只在环上有一个点，数据可能分布不均。

虚拟节点做法：

```text
nodeA#1
nodeA#2
nodeA#3
```

每台真实机器映射多个虚拟节点，让数据分布更均匀。

## 限流算法

### 1. 固定窗口

按固定时间窗口计数。

```text
每分钟最多 1000 次
```

优点：

- 简单。

缺点：

- 边界突刺明显。

例如 12:00:59 来 1000 次，12:01:00 又来 1000 次，短时间内等于 2000 次。

### 2. 滑动窗口

把时间窗口拆成多个小格子，统计最近一段时间的请求量。

优点：

- 比固定窗口平滑。

缺点：

- 实现复杂一些。
- 需要维护多个桶。

### 3. 漏桶

请求像水一样进入桶，系统以固定速率处理。

```text
进水速度不固定
出水速度固定
桶满则丢弃
```

特点：

- 输出平滑。
- 适合保护下游稳定处理。

缺点：

- 突发流量处理能力弱。

### 4. 令牌桶

系统按固定速率生成令牌，请求必须拿到令牌才能通过。

```text
有令牌 -> 放行
无令牌 -> 拒绝或等待
```

特点：

- 支持一定突发流量。
- 工程中很常用。

适合：

- API 限流。
- 用户限流。
- 商家限流。
- 下游调用限流。

## 限流维度

跨境物流系统常见：

- 单 IP。
- 单用户。
- 单商家 tenant。
- 单接口。
- 单承运商。
- 单仓库。
- 全局系统容量。

例如开放平台要按商家限流，防止一个大商家或异常调用影响其他租户。

## 消息重试机制

### 为什么需要重试

分布式系统里很多失败是短暂的：

- 网络抖动。
- 下游临时超时。
- 数据库瞬时锁冲突。
- 第三方承运商接口偶发失败。

这些可以通过重试恢复。

## 重试设计原则

### 1. 必须幂等

重复消费不能导致重复扣库存、重复创建订单、重复退款。

常见幂等方式：

- event_id 去重。
- request_id 去重。
- 数据库唯一索引。
- 状态机判断。

### 2. 限制重试次数

不能无限重试。

```text
最多重试 3 次
```

多次失败后进入死信队列。

### 3. 退避策略

不要失败后立刻高频重试。

常见：

- 固定间隔。
- 指数退避。
- 指数退避加随机抖动。

示例：

```text
1s -> 2s -> 4s -> 8s
```

### 4. 区分错误类型

不是所有错误都应该重试。

可以重试：

- 网络超时。
- 5xx。
- 临时限流。

不应该重试：

- 参数错误。
- 权限错误。
- 业务状态不允许。
- 库存不足。

### 5. 死信队列

超过重试次数仍失败，进入 DLQ，等待人工处理或补偿任务。

## 面试回答模板

一致性哈希主要解决节点扩缩容时大量 key 重映射的问题，常用于分布式缓存、分片路由和负载均衡。通过 hash 环和虚拟节点，可以减少迁移范围并让数据分布更均匀。

限流算法常见有固定窗口、滑动窗口、漏桶和令牌桶。固定窗口简单但有边界突刺；滑动窗口更平滑；漏桶固定速率输出；令牌桶既能限速又能允许一定突发流量，所以 API 网关和服务限流里很常用。

消息重试用于处理临时失败，但不能无限重试。要配合幂等、最大次数、退避、错误分类和死信队列。像订单、库存、支付这类非幂等业务，必须先保证重复消息不会产生副作用，才能安全重试。

## 常见追问

### 1. 一致性哈希为什么需要虚拟节点？

为了让数据分布更均匀。如果真实节点数量少，环上分布可能不均，虚拟节点可以降低倾斜。

### 2. 令牌桶和漏桶区别是什么？

漏桶以固定速率处理请求，输出更平滑；令牌桶按固定速率生成令牌，但可以积累令牌，所以允许一定突发流量。

### 3. 消息重试为什么要加随机抖动？

防止大量失败请求在同一时间一起重试，造成重试风暴。
